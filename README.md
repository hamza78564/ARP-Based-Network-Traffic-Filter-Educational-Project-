# ARP-Based Network Traffic Filter (Educational Project)

## Overview

This project is an educational cybersecurity lab that demonstrates how ARP spoofing and packet filtering work in a controlled environment.

The script performs ARP poisoning between a target device and the gateway, forwards intercepted traffic, and drops packets that match a configurable blacklist. It was created to better understand network protocols, packet inspection, and defensive security concepts.

> **This project is intended for educational purposes only and should only be used in environments where you have explicit authorization.**

---
**Development Notes
**
To better understand how network filtering works, I first captured network traffic in a controlled lab environment using packet sniffing to identify the domain names and DNS requests generated by a web browser and the TikTok application.

After identifying the relevant domains, I implemented DNS and packet filtering rules to block traffic associated with those domains.

Results
Web Browser: Successfully blocked access by filtering the identified domains.
TikTok Application: The same approach was only partially effective. While some comments and video requests were blocked, the application continued to function in many cases.

This experiment demonstrated that modern applications often rely on multiple domains, encrypted connections, CDNs, and additional networking techniques, making simple domain-based filtering insufficient for completely blocking application functionality.

This project significantly improved my understanding of packet analysis, DNS traffic, ARP spoofing, and network traffic filtering.

## How It Works

1. Enables IP forwarding.
2. Inserts an iptables rule to send forwarded packets to Netfilter Queue.
3. Performs ARP spoofing between the target and the gateway.
4. Intercepts forwarded packets.
5. Inspects DNS queries and packet payloads.
6. Drops packets matching blacklist entries.
7. Allows all other traffic to continue normally.
